## Notes

### Solutions serving multiple services (canonical home chosen; other touched services listed)

- **Keycloak** (canonical: security/access-control-and-trust/identity-provider-federation) also anchors the Tier 1 Authentication Provider service and is consumed by participant-management/user-roles for user storage and by participant-management/onboarding for temporary credential issuance.
- **Spring Cloud Gateway** (canonical: security/access-control-and-trust/authorisation) also implements the catalogue Query Mapper Adapter (resource-discovery/search-engine) and the Tier 1/Tier 2 gateways in access-control-and-trust/authorisation (both RBAC and ABAC).
- **EJBCA** (canonical: security/access-control-and-trust/identity-provider-federation) also underpins credential signing activities related to security/credential-management/signing.
- **Eclipse Dataspace Connector (EDC)** (canonical: integration/resource-sharing/resource-sharing-runtime) also hosts the Policy Engine (integration/policy-enforcement/policy-enforcement-point), the S3 data-plane extension (integration/data-sharing/bulk-data-transfer), the Triggering Extension (infrastructure/provisioning/infrastructure-provisioning), and the Local Assets Catalogue (integration/resource-discovery/resource-catalogue).
- **XFSC Organisational Credential Manager (OCM)** (canonical: security/credential-management/wallet) also implements the SD Creation Tool (governance/resource-management/metadata-description), the Policy Template Datastore (governance/policy-management/policy-administration-point), and the Signer Service (security/credential-management/signing).
- **Apache Kafka** (canonical: administration/notification-and-messaging/notification) also acts as message broker for contract manager async flows (governance/contract-management/contract-establishment) and infrastructure provisioning triggering (infrastructure/provisioning/infrastructure-provisioning).
- **PostgreSQL** (canonical: foundations/data-architecture) is pervasive — persistence layer for Keycloak, EJBCA, identity/onboarding/user-roles databases, catalogue metadata store, infrastructure provider storage, and Spring Cloud Gateway.
- **HashiCorp Vault / OpenBao** (canonical: security/access-control-and-trust/encryption) also stores credentials for Tier 2 Authentication Provider (security/access-control-and-trust/authentication-provider-federation).
- **Argo CD** (canonical: infrastructure/supporting-infrastructure-services/infrastructure-orchestration) is also the deployment tool for DevSecOps pipelines (foundations/devsecops) and implements Infrastructure Provisioner (infrastructure/provisioning/infrastructure-provisioning).
- **Apache Jena Fuseki** (canonical: data/semantics-and-vocabulary/schema-management) is also used as catalogue RDF triple-store alternative (integration/resource-discovery/resource-catalogue).
- **Neo4j** (canonical: integration/resource-discovery/resource-catalogue) is used across schema-management graph queries (data/semantics-and-vocabulary/schema-management).
- **ELK stack** (canonical: administration/observability/logging) also feeds reporting (administration/observability/reporting) and is reused for audit (governance/audit) — e.g. business logs are indexed in Elastic for audit trails.
- **Schema Management Service (SMS)** (canonical: data/semantics-and-vocabulary/schema-management) also surfaces in governance/resource-management/metadata-description (as the Governance Authority's metadata description BB).
- **Gitea** (canonical: foundations/devsecops) is also used as the Git-based Repository for Infrastructure Provider Storage (infrastructure/provisioning/infrastructure-provisioning) and for the Dagster code location repository (data/supporting-data-services/data-orchestration).
- **Crossplane** (canonical: infrastructure/provisioning/infrastructure-provisioning) is also invoked by the Infrastructure Provisioner (same service) and referenced in §6.5.2 as the backing deployment-script engine.
