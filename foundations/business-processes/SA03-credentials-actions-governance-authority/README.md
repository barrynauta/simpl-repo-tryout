# SA03 – Credentials actions by the Governance Authority

## Overview

The Governance Authority manages the lifecycle of Participant credentials. Credentials ensure access rights align with data space rules. The concept of **mandates** is used to provide official approval for actions.

## Credential Actions

-   **Revoking:** Permanently blocks Participant actions.
-   **Suspending:** Temporarily halts actions.
-   **Re-activating:** Restores functionality to suspended credentials.
-   **Renewing:** Issuance of new credentials.
-   **Editing Identity Attributes:** Updates to permissions or status.

## Actors

-   **Governance Authority Representative:** Manages the credentials.
-   **Participant Representative:** Administers agent configuration.

## Detailed Steps

| Step ID | Action | Description |
| :--- | :--- | :--- |
| **SA03.01** | **Revoke** | Permanently block Participant actions. |
| **SA03.02** | **Suspend** | Temporarily block actions. |
| **SA03.03** | **Re-activate** | Restore functionality. |
| **SA03.04** | **Edit Attributes** | Update identity attributes/permissions. |
| **SA03.05** | **Renew** | Issue new credentials for the Participant. |

## Outcomes

-   **Credential State Changed:** The Participant's access is updated (revoked, suspended, or active).
