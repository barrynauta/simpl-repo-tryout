# SD Schemas

Default Self-Description schemas (data, infrastructure, application) used as samples and as fixtures for integration testing the catalogue, schema-management, and validation flows. Generated from YAML via [`sd-schemas-util`](../../utils/sd-schemas-util/README.md).

The PSO mapping spreadsheet flags these schemas for *extraction* from the as-is repo so the runtime utility (`sd-schemas-util`) and the fixture data (here) live in separate folders.

Provenance: built by Simpl. Source repository (currently bundled): `data1/sdtooling-sd-schemas`. Owner: Data1 team. Licence: EUPL 1.2.
