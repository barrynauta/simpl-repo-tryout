# eck-monitoring (reference)

> **This is a cross-service reference.** The canonical solution folder lives at [`administration/observability/performance-monitoring/eck-monitoring/`](../../performance-monitoring/eck-monitoring/README.md).

ECK manages the Elasticsearch + Kibana cluster that the [ELK stack](../../) uses to store and query logs. The same Elasticsearch instance also stores metrics from Metricbeat, Heartbeat, and Prometheus, which is why ECK is documented under the **Performance monitoring** service rather than duplicated here.

For the canonical URL, deployment guide, and architecture, follow the link above.
